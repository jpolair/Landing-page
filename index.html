Développement Front-End

Pour le développement Front-End j'utilise les langages stables les plus modernes et les plus utilisés à ce jour c'est-à-dire le HTML5, le CSS3 et bien sur le JavaScript.Pour leur richesse et leur flexibilité j'utilise les frameworks CSS Bootstrap et Materialize.Pour une expérience utilisateur incomparable j'utilise le framework MVVM AngularJs ( il est formidable ) L'arrivée des components ( après la version 1.5 ) a redonné un nouvel élan à ce framework qui, à mon avis, sera de plus en présent ces prochaines années.Angular est super mais c'est une erreur de ne pas s'intéresser à ce qui se passe du côté des autres framework JavaScript c'est la raison pour laquelle je me suis promis d'aller regarder ailleurs ce qu'il se passe, notamment du côté de React et surtout de Vue.Js.....

Développement Back-End

Pour le développement Back-End j'utilise également des technologies très modernes, je crois en la puissance du JavaScript côté serveur : j'utilise donc NodeJS ( avec ExpressJs bien sûr ).La puissance du moteur V8 de Google mise sur un serveur permet de développer des applications très rapides et modifie considérablement l'expérience utilisateur.En ce qui concerne le système de gestion de base de données pour les gros volumes et une gestion souple je crois en la puissance du NoSQL grâce à MongoDB, toutefois, quand le NoSQL n'est pas adapté au projet ,ou, pour des plus petits projets rien de tel qu'un SGBDR tel que MySQL qui n'a plus rien à prouvé.Autre chose, plus je lis des articles, plus je me dis qu'il va falloir s'intéresser au langage d'une petite société américaine qui monte, c'est le Go.... cette petite société dit que les machines ont beaucoup évoluées et que les langages utilisés aujourd'hui ne sont pas optimisés pour nos ordis super puissant, il va falloir que je creuse un peu de se côté là....

Outils

Nous sommes en 2017, pour travailler dans de bonnes conditions il faut les bons outils, j'ai pour habitude de les tester et de lire les retours des communautés de développeurs pour choisir les bons.Aujoud'hui pour versionner mon code je me sert de git/github, de gulp pour automatiser mes tâches , d'un environnement Unix avec un shell zsh pour travailler dans mon terminal, et après avoir testé Atom, sublime-text, Brackets, mon choix s'est arreté, pour l'instant, sur Visual Studio Code, ainsi je participe à l'entrée de Microsoft dans l'open source :-)
